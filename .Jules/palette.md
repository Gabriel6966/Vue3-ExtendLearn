## 2026-04-22 - Missing ARIA Labels on Form Elements Bound with v-model in Vue
**Learning:** When using `v-model` with native HTML form inputs (like `<input>`, `<select>`) in Vue components, these elements do not automatically receive accessible labels or `aria-label`s. Screen reader users can struggle to understand the purpose of these form elements if they only have placeholder text or lack visual labels. Furthermore, inline styles for properties like `text-decoration` must be correctly spelled (`line-through` vs `line-trought`) to avoid silent CSS failures in dynamically bound `:style` attributes.
**Action:** Ensure all native form elements (inputs, selects, checkboxes) bound to `v-model` are given explicit `aria-label` attributes if there's no `<label>` element directly associating text. Always review inline CSS string values for correct spelling as Vue will not throw a runtime error for invalid properties.
